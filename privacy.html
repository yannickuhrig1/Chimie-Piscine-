Chimie Piscine — Confidentialité

Politique de confidentialité

Dernière mise à jour : 30 mai 2026 — Version de l'application : 1.22.1

En résumé : Chimie Piscine fonctionne par défaut sans aucun compte, et stocke vos mesures uniquement dans votre navigateur ou votre téléphone. La création d'un compte (par e-mail) est optionnelle et sert uniquement à synchroniser vos données entre vos appareils. Aucune publicité, aucun revendeur tiers, aucun partage avec des courtiers de données.

1. Qui est responsable du traitement ?

Yannick Uhrig, éditeur indépendant de l'application Chimie Piscine (PWA et version Android), accessible à chimie-piscine.vercel.app.

Contact pour toute question liée à vos données : support@yannick-uhrig.com.

2. Comment l'application fonctionne par défaut

Au premier lancement, l'application fonctionne entièrement en local. Vos mesures (pH, chlore, TAC, etc.), la configuration de vos bassins et vos préférences sont stockés dans le localStorage de votre navigateur ou dans le stockage interne de l'application Android. Rien n'est envoyé sur un serveur, à l'exception des appels de météo si vous activez cette fonction (voir §5).

Vous pouvez à tout moment effacer ces données depuis les réglages de votre navigateur (effacer les données du site) ou en désinstallant l'application Android.

3. Création d'un compte (optionnelle)

Si vous choisissez de créer un compte, l'application utilise Supabase pour stocker une copie de vos données dans le cloud afin de les synchroniser entre vos appareils.

Données collectées au moment de la connexion

  • Votre adresse e-mail, utilisée comme identifiant de connexion via un lien magique (pas de mot de passe à retenir).
  • Un identifiant utilisateur technique (UUID) généré par Supabase.

Données synchronisées si vous êtes connecté

  • La configuration de vos bassins (nom, volume, désinfection, ville, coordonnées si renseignées).
  • Vos mesures de chimie (date, pH, chlore, TAC, CYA, sel, TH, phosphates, brome, température, notes).
  • Vos rappels et préférences d'affichage (thème, métriques visibles).

Confidentialité technique

Les tables Supabase sont protégées par des règles Row-Level Security strictes : seul le propriétaire d'un enregistrement peut le lire ou le modifier. Personne d'autre, y compris l'éditeur de l'application, ne peut accéder à vos données via l'interface publique.

4. Sous-traitants et services tiers

Vercel Hébergement

Héberge la version web de l'application. Vercel reçoit votre adresse IP et votre user-agent dans le cadre standard d'une visite HTTP. Politique Vercel.

Supabase Base de données — compte opt-in

Fournit l'authentification par lien magique et le stockage cloud de vos données si vous créez un compte. Données stockées en UE. Politique Supabase.

Resend E-mails transactionnels

Envoie les e-mails de connexion (lien magique) et les éventuelles réponses aux tickets de support. Votre adresse e-mail est transmise pour l'envoi uniquement. Politique Resend.

Chatwoot (auto-hébergé) Tickets de support

Si vous utilisez le formulaire de contact intégré (page /#contact), votre nom, votre e-mail et votre message sont enregistrés dans une instance Chatwoot auto-hébergée sur l'infrastructure de l'éditeur (UE). Aucun cookie tiers.

Umami (auto-hébergé) Statistiques anonymes

Mesure d'audience respectueuse de la vie privée, sans cookie, sans empreinte navigateur, avec IP anonymisée. Aucun identifiant personnel n'est collecté. Instance hébergée par l'éditeur (UE). Détails Umami.

Open-Meteo Météo

Si vous activez la météo locale, l'application envoie à api.open-meteo.com soit les coordonnées GPS de votre bassin (que vous avez vous-même renseignées), soit un nom de ville / code postal pour la fonction de recherche. Service gratuit, sans inscription. Conditions Open-Meteo.

Cloudflare Réseau / Proxy

Sert de relais pour certains sous-domaines auto-hébergés (tickets, statistiques). En tant que CDN, Cloudflare voit transiter votre adresse IP. Politique Cloudflare.

5. Permissions sensibles

Géolocalisation

Demandée uniquement si vous appuyez sur le bouton « Utiliser ma position » lors de la configuration d'un bassin, pour pré-remplir vos coordonnées et activer la météo locale. La position n'est ni envoyée à un serveur de l'éditeur, ni stockée à votre insu : elle est uniquement utilisée localement, puis (si vous validez) sauvegardée dans la configuration du bassin que vous possédez.

Notifications

Si vous activez les rappels avec notifications, votre navigateur génère un identifiant d'abonnement push (endpoint chiffré). Cet identifiant est stocké côté Supabase pour permettre l'envoi des rappels. Aucun contenu personnel n'est inclus dans les notifications au-delà des intitulés que vous avez configurés (ex : « Mesurer le chlore »).

Stockage local

L'application utilise localStorage et le cache du service worker pour fonctionner hors-ligne. Vous pouvez les effacer à tout moment depuis votre navigateur ou les paramètres système.

7. Durée de conservation

  • Données locales : conservées tant que vous ne les effacez pas vous-même.
  • Données de compte Supabase : conservées tant que votre compte existe. Suppression complète sur simple demande par e-mail.
  • Tickets de support Chatwoot : conservés 2 ans après la dernière interaction, puis archivés ou supprimés.
  • Statistiques Umami : aucune donnée individuelle, agrégats conservés sans limite.

8. Sécurité

Les échanges réseau sont chiffrés (HTTPS / TLS) de bout en bout. Les bases de données Supabase utilisent Row-Level Security avec authentification par JWT signé. Les clés sensibles (clés API SMTP, clés VAPID privées) ne quittent jamais l'infrastructure de l'éditeur.

9. Cookies et traceurs

L'application n'utilise aucun cookie publicitaire. Elle utilise uniquement :

  • Un cookie de session technique uniquement si vous êtes connecté à un compte (token Supabase, expiration 1 h, renouvelé automatiquement).
  • Le stockage local (localStorage) pour conserver vos données entre les sessions.

Umami fonctionne sans cookie.

10. Enfants

L'application n'est pas conçue pour les enfants de moins de 13 ans et ne collecte volontairement aucune donnée les concernant.

11. Modifications

Cette politique peut évoluer en cas d'ajout de fonctionnalité. La date de mise à jour ci-dessus reflète la version en vigueur. Les modifications substantielles seront annoncées dans les notes de version au lancement de l'application.
